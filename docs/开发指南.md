# 开发指南

更新日期：2026-04-29

## 1. 运行环境

- Node.js 18+
- npm 9+
- PostgreSQL 14+

## 2. 初始化步骤

```bash
npm install
copy .env.local.example .env.local
npm run db-generate
npm run db-migrate
npm run dev
```

## 3. 常用命令

```bash
npm run dev
npm run build
npm run start
npm run lint
npm run type-check
npm run format:write
npm run db-generate
npm run db-migrate
```

## 4. 本地模式约定

### 4.1 Bootstrap

首次启动或访问本地 API 时，会自动创建：

- 固定本地用户
- 固定本地 Profile
- 默认工作区

对应实现见：

- `lib/local-config.ts`
- `lib/project-config.ts`
- `lib/local-bootstrap.ts`

### 4.2 本地存储

- 默认目录：`storage/`
- 可通过 `LOCAL_STORAGE_PATH` 覆盖
- `storage/` 视为运行期数据，不应作为源码提交

## 5. 改代码时的标准流程

### 5.1 新增一个已支持实体的接口

1. 先更新 `prisma/schema.prisma`
2. 再实现 `app/api/local/*` 路由
3. 再实现对应 `db/*` 包装层
4. 再补 UI 调用
5. 最后更新文档：
   - `docs/API说明.md`
   - `docs/能力边界.md`
   - 如有必要，更新 `README.md`

### 5.2 新增/改造知识层能力（Knowledge Agent）

按以下顺序落地，避免“只有 UI/文档，没有服务端”的漂移：

1. 先改 `prisma/schema.prisma`（`vaults/notes/note_chunks/note_links/retrieval_events`）
2. 再补 `app/api/knowledge/*` 路由
3. 再补 `lib/agent/*`、`lib/knowledge/*`、`lib/obsidian/*` 编排与运行时模块
4. 最后接 UI 渐进披露（Sources / Drawer / Debug）

### 5.3 类型约束约定

- `types/database.ts` 已收敛为“本仓库表名白名单 + 宽结构过渡类型”
- 严禁再回到 Supabase 全量类型镜像
- 新增表时同步扩充 `LocalTableName`，并按需为该表补精确字段类型

### 5.2 遇到未落地能力

如果后端尚未落地：

- 读操作：
  - 工作区初始化需要的列表读取，可返回显式空集合
  - 其余读取应避免假装存在真实数据
- 写操作：
  - 必须显式抛错
  - 禁止返回 `null`、`[]`、`true` 等伪成功值

### 5.3 修改文件上传或检索链路

同时检查：

- `app/api/local/files/route.ts`
- `lib/retrieval/processing/*`
- `app/api/memory/retrieve/route.ts`
- `storage/` 路径拼接与越界风险

## 6. 提交前检查

至少执行：

```bash
npm run lint
npm run type-check
```

如改动涉及 Prisma：

```bash
npm run db-generate
```
