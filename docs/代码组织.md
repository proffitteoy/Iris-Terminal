# 代码组织

更新日期：2026-04-29

本文档只描述当前仓库的实际代码组织规则，不再停留在通用原则层面。

## 1. 顶层目录职责

### `app/`

- 页面与 API 路由入口
- 只放页面装配、路由参数解析和 Route Handler
- 不在这里堆积可复用业务逻辑

### `components/`

- React 组件
- `chat/`、`sidebar/`、`messages/` 等子目录按 UI 领域拆分
- 复杂副作用尽量下沉到 hooks 或 `lib/`

### `context/`

- 全局 UI 状态定义
- 只存放跨页面共享的前端状态，不承载服务端业务逻辑

### `db/`

- 前端数据访问层
- 负责调用 `/api/local/*` 或显式抛出不支持错误
- 禁止继续保留 `null/[]/true` 伪实现

### `lib/`

- 服务端与通用逻辑
- 包含：
  - bootstrap
  - 本地存储
  - 向量化与检索
  - 配置与工具函数

### `prisma/`

- 本地模式真实数据库模型
- 所有“已支持实体”都必须能在这里找到对应定义

### `types/`

- 共享类型
- 当前 `types/database.ts` 存在历史遗留类型面偏大问题，修改时必须与 Prisma schema 对照

## 2. 当前推荐依赖方向

```text
UI 组件
  ↓
db/*
  ↓
app/api/local/*
  ↓
lib/* / prisma / storage
```

约束：

- `components/*` 不直接调用 Prisma
- `db/*` 不自行实现持久化逻辑
- `app/api/*` 不承载复杂前端状态

## 3. 文件落点规则

### 新增本地数据读写能力

- 页面入口：`app/api/local/<entity>/route.ts`
- 前端调用：`db/<entity>.ts`
- 共享逻辑：`lib/`
- 文档同步：`docs/API说明.md`、`docs/能力边界.md`

### 新增仅前端可复用逻辑

- Hook：`lib/hooks/`
- 纯函数工具：`lib/`
- UI 组件：`components/`

## 4. 当前重构后的组织原则

- 支持能力：真实实现
- 未支持能力：显式失败
- 列表型禁用能力：只在工作区初始化链路中返回空集合
- 任何写路径都不能再伪装成功

## 5. 代码评审重点

- 是否把新逻辑放到了正确层
- 是否新增了没有 Route Handler 支撑的 `db/*` 写接口
- 是否又引入了“来源即可信”的参数直传
- 是否同步更新了文档和能力边界
